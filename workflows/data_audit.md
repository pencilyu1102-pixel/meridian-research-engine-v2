# Data Audit Workflow

## Purpose

Determine whether data can support a formal research conclusion.

## Source Tiers

| Tier | Description |
|---|---|
| S | Official filing, official release, regulator source |
| A | Direct company material or verified exchange data |
| B | Reputable financial data vendor or major media with clear sourcing |
| C | Secondary commentary, estimated data, or unclear transformation |
| D | Unverified social media, stale screenshot, broken link, or unknown source |

## Steps

1. List all key data points.
2. Record source, timestamp, unit, currency, and accounting basis.
3. Identify conflicts and stale data.
4. Decide whether each data point can enter the conclusion.
5. Downgrade the report if material data fails the audit.

## Output

- Data point cards.
- Conflict list.
- Report confidence level.
- Whether account-level suggestions are allowed.
