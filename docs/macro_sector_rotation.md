# Macro & Sector Rotation Engine

宏观与板块轮动引擎用于把宏观共识、市场价格验证和板块景气度工程化。它不是预测系统，不输出公开买卖建议，也不把某个板块的短期强弱直接转换成交易指令。

核心一句话：

> 用多家权威机构的共识和分歧，结合市场价格验证，把宏观环境翻译成可审计、可反驳的板块轮动判断。

## 1. 适用范围

- 判断宏观环境对股票市场的顺风、逆风或中性影响。
- 分析不同市场和板块的轮动状态。
- 识别机构共识、机构分歧和边际变化。
- 判断主流共识是否已经被市场价格、估值和仓位反映。
- 为行业位置、公司基本盘和估值分析提供上游背景。

## 2. 不适用范围

- 不基于单一机构报告形成结论。
- 不把宏观观点直接变成个股买卖建议。
- 不展示真实个人持仓、真实成本或真实加减仓动作。
- 不用整数点位、主题热度或媒体叙事生成操作信号。

## 3. 三类证据来源

宏观判断至少覆盖三类来源，并保留冲突。

| 层级 | 来源类型 | 用途 |
|---|---|---|
| 第一层 | 官方或准官方来源 | 判断政策、增长、通胀、信用和流动性的大方向 |
| 第二层 | 主流资管、投行、研究机构 | 判断机构资金如何理解和交易宏观 |
| 第三层 | 市场价格和高频指标 | 验证机构观点是否已经被资金行为反映 |

常见市场价格指标包括：利率、实际利率、期限利差、美元、信用利差、VIX、主要指数、行业 ETF、宽度、成交量、资金流和估值分位。

## 4. 六因子宏观评分

每次正式宏观分析必须围绕六个因子展开。

| 因子 | 核心问题 | 主要影响 |
|---|---|---|
| 增长 | 经济是在加速、放缓还是分化？ | 顺周期、防御板块、盈利预期 |
| 通胀 | 通胀是上行、下行还是粘性？ | 利率、资源品、消费、估值容忍度 |
| 流动性 | 央行是在宽松、中性还是收紧？ | 成长股估值、信用条件、风险偏好 |
| 信用 | 企业和居民融资是在扩张还是收缩？ | 银行、地产、周期品、消费信心 |
| 盈利 | 企业盈利预期是在上修还是下修？ | 股票市场核心驱动力 |
| 风险偏好 | 市场愿意冒险还是避险？ | 科技、小盘、高估值、防御资产 |

每个因子给出 -2 到 +2 分：

| 分数 | 含义 |
|---:|---|
| +2 | 明显利好风险资产 |
| +1 | 温和利好风险资产 |
| 0 | 中性、分歧较大或证据不足 |
| -1 | 温和利空风险资产 |
| -2 | 明显利空风险资产 |

综合分公式：

```text
宏观综合分 = 增长分 + 通胀分 + 流动性分 + 信用分 + 盈利分 + 风险偏好分
```

标准化分数：

```text
标准化分数 = (宏观综合分 + 12) / 24 * 100
```

| 标准化分数 | 结论 |
|---:|---|
| 80-100 | 宏观强顺风 |
| 65-80 | 宏观偏顺风 |
| 50-65 | 中性偏观察 |
| 35-50 | 宏观偏逆风 |
| 35 以下 | 宏观强逆风 |

## 5. 信息源优先级与来源权重

正式报告中，宏观观点卡片应记录来源类型、新鲜度和证据强度。

| 来源类型 | 参考权重 |
|---|---:|
| 央行 / IMF / OECD / 世界银行 / BIS | 1.3 |
| 国家统计局 / 财政部 / 官方数据 | 1.2 |
| 大型资管 / 长周期资产配置研究 | 1.0 |
| 投行策略报告 | 0.8 |
| 媒体报道 / 市场传闻 | 0.4 |

| 信息发布时间 | 新鲜度权重 |
|---|---:|
| 7 天内 | 1.0 |
| 1 个月内 | 0.8 |
| 1-3 个月 | 0.6 |
| 3 个月以上 | 0.3 |

| 证据情况 | 证据强度权重 |
|---|---:|
| 数据、政策、市场价格同时验证 | 1.2 |
| 数据和政策验证 | 1.0 |
| 只有机构观点 | 0.7 |
| 只有新闻叙事 | 0.4 |

单条观点参考公式：

```text
单条观点得分 = 方向分 * 来源权重 * 新鲜度权重 * 证据强度权重
```

## 6. 市场价格验证

机构报告代表观点，市场价格代表资金行为。每轮宏观分析必须检查机构共识是否已经被市场交易。

| 步骤 | 检查问题 |
|---:|---|
| 1 | 从观点卡片中提取机构一致看好的资产或板块 |
| 2 | 检查相关资产或板块价格是否已先行 |
| 3 | 检查估值分位、成交量、资金流和仓位是否拥挤 |
| 4 | 判断定价状态：已充分反映、部分反映、尚未充分反映、可能过度反映 |
| 5 | 说明研究意义：是共识交易、预期差，还是需要等待新催化 |

定价状态定义：

| 状态 | 定义 | 研究含义 |
|---|---|---|
| 已充分反映 | 价格、估值、仓位大多已经体现共识 | 不把共识本身当作新增优势 |
| 部分反映 | 价格已有反应，但估值或仓位仍未极端 | 继续跟踪催化和拥挤度 |
| 尚未充分反映 | 共识较强但价格、估值、仓位未充分体现 | 可能存在预期差，但仍需行业和估值验证 |
| 可能过度反映 | 价格涨幅显著领先基本面改善 | 降低结论强度，等待复盘触发 |

## 7. 三道门原则

一个板块不能只因为宏观顺风就进入重点研究，必须通过三道门。

| 门槛 | 判断问题 | 未通过时的处理 |
|---|---|---|
| 第一门：宏观风向 | 宏观是否支持该板块？ | 只保留观察，不形成强结论 |
| 第二门：行业景气 | 订单、价格、库存、利润率是否改善？ | 等待行业数据验证 |
| 第三门：估值与交易拥挤度 | 估值是否合理？交易是否拥挤？ | 防止追随已经过热的共识 |

三道门全部通过，也只代表该板块进入更高优先级研究，不代表公开买卖建议。个股层面仍需进入公司基本盘、估值、安全边际、持仓执行和 Bear Case。

## 8. A 股、港股、美股差异化处理

同一套宏观评分不能机械套用到所有市场。Macro & Sector Rotation Engine 必须先确认市场结构，再解释宏观因子的传导路径。

| 市场 | 重点变量 | 解释原则 |
|---|---|---|
| A 股 | 国内政策、信用周期、人民币流动性、行业政策、北向与公募资金行为 | 更重视政策边际、信用扩张和行业景气验证，避免只用海外利率解释本土板块 |
| 港股 | 美元利率、人民币预期、南向资金、互联网与金融权重、流动性折价 | 同时看中国基本面和海外折现率，区分基本面改善与估值修复交易 |
| 美股 | Fed 路径、实际利率、美元、盈利修正、ETF 资金流和市场宽度 | 更重视利率和盈利预期对估值容忍度的影响，并检查拥挤交易是否已经 price in |

跨市场比较时，必须说明数据口径、货币环境、投资者结构和估值体系差异。不能因为一个市场的宏观结论顺风，就直接推导另一个市场或个股的操作动作。

## 9. 周 / 月 / 季工作流

| 频率 | 主要任务 | 输出 |
|---|---|---|
| 每周 | 更新市场价格、高频指标、资金流、ETF 流入、相对强弱和拥挤度 | 判断机构共识是否被市场价格进一步反映，更新观察清单 |
| 每月 | 更新官方宏观数据、通胀、就业、PMI、信用、盈利修正和行业景气 | 重新计算六因子宏观评分，调整顺风 / 逆风板块判断 |
| 每季度 | 复盘宏观假设、机构共识命中率、市场定价偏差和三道门有效性 | 保留、降级或推翻原有宏观与板块轮动框架 |

周度工作流更偏验证价格和拥挤度，月度工作流更偏验证数据和政策，季度工作流更偏复盘方法是否失效。任何频率下都不得把宏观顺风直接写成公开买卖建议。

## 10. 输出结构

宏观与板块轮动报告应包含：

1. 一句话结论；
2. 机构共识与分歧；
3. 六因子宏观评分；
4. 宏观状态识别；
5. 市场价格是否已反映机构共识；
6. 板块轮动地图；
7. 三道门检查；
8. 重点顺风和逆风板块；
9. 风险清单；
10. 后续跟踪指标；
11. 失效条件。

## 11. 最简执行流程

时间有限时，至少完成：

1. 获取三类来源：至少 1 个官方或准官方来源、2 个主流机构来源、3 个市场价格或高频指标。
2. 判断六大宏观因子。
3. 识别当前宏观状态。
4. 识别顺风和逆风板块。
5. 判断市场是否已经反映共识。
6. 输出风险、后续跟踪指标和失效条件。

## 12. 与其他模块的接口

本引擎输出进入以下模块：

- `Industry Position Agent`：验证宏观是否传导到行业景气；
- `Company Fundamental Agent`：验证公司是否能兑现行业 beta 或自身 alpha；
- `Valuation & Margin Agent`：检查估值是否已经反映宏观和行业共识；
- `Price Level Engine`：只生成研究地图，不输出自动交易信号；
- `Report Gatekeeper`：检查宏观、行业、估值和执行条件是否完整。
