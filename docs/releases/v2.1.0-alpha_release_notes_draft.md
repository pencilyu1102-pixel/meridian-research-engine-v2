# v2.1.0-alpha — Data Integrity Hardlock  
# v2.1.0-alpha — 数据完整性硬锁

## English

This is an alpha prerelease of **Meridian Research Engine 2.0**.

The main purpose of this release is to upgrade the engine from **structure-level report gating** to **data-integrity-based release gating**.

A report is no longer considered formally releasable simply because its structure is complete. Formal release eligibility now requires both:

- `Data Integrity Hardlock = PASS_FORMAL`
- `Gatekeeper structure check = FULL_PASS`

## What changed

### 1. Data Card Registry

Added a Data Card schema and registry MVP to ensure that key fields used in conclusions have explicit provenance, period, unit, currency, source tier, timestamp, and conclusion permission.

Principle:

> No card, no conclusion.

### 2. Earnings Basis Hardlock

Added earnings-basis checks to prevent unsupported mixing of GAAP actuals, adjusted figures, guidance, and aggregator fields.

This reduces the risk that reference-only or forward-looking figures are incorrectly used as core valuation denominators.

### 3. Capital Structure Bridge

Added capital-structure bridge checks for share-count, market-cap, EPS, and PE consistency.

This helps detect cases where current share count, weighted diluted shares, market cap, EPS, and PE are not reconciled.

### 4. Data Integrity Hardlock Aggregator

Added a hardlock aggregation layer that combines module-level verdicts into a formal release decision.

If any critical hardlock module fails, the report cannot be formally released.

### 5. Gatekeeper Integration

Integrated Gatekeeper with Data Integrity Hardlock verdicts.

A structure-only `FULL_PASS` is no longer sufficient for formal release. If the hardlock verdict is missing or failing, the report is downgraded or blocked from formal release.

### 6. Valuation Metadata Guard

Valuation tools can still calculate with bare numeric inputs, but such outputs are explicitly downgraded to:

```text
CALC_ONLY_NOT_FOR_CONCLUSION
```

Only inputs with complete and approved metadata can enter the formal conclusion chain.

### 7. Industry Hard Fields MVP

Added Industry Hard Fields MVP with two initial adapters:

- `CN_SEMICONDUCTOR_GROWTH`
- `US_MANAGED_CARE`

Industry-specific hard fields must be present and eligible for formal conclusion use. Missing, blocked, or reference-only fields prevent formal release.

The managed-care adapter was generalized before release from company-specific segment names to generic managed-care segment fields.

## Validation

Final validation completed before release:

- Full test suite: `59 passed`
- Focused hardlock regression suite: `42 passed`
- Public boundary grep: `0 hits`
- `reports/` contains SAMPLE-only files
- No real-company sample reports were added
- No real price ranges, valuation conclusions, or trading recommendations were added

## Public boundary

This release does not include:

- real-company public sample reports
- real price ranges
- valuation conclusions
- trading suggestions
- account-level actions
- private validation materials

Public examples remain synthetic and SAMPLE-only.

## Known limitations

This is still an alpha release.

Known limitations:

- hardlock inputs currently rely on sidecar JSON
- markdown body parsing is not yet implemented
- no automatic market or financial data fetching
- no full industry database
- industry adapters are MVP-level and will need expansion
- this release validates release-gating mechanics, not end-to-end investment automation

## Release status

This release should be treated as a prerelease.

It is suitable for internal validation, further framework testing, and controlled iteration. It is not a production-grade investment recommendation system.

## 中文说明

这是 **Meridian Research Engine 2.0｜经纬投研引擎 2.0** 的 alpha 预发布版本。

本次发布的核心目标，是将引擎从 **"报告结构准出"** 升级为 **"数据口径准出"**。

也就是说，一份报告不能再因为结构完整就被视为正式可准出。正式准出必须同时满足：

- `Data Integrity Hardlock = PASS_FORMAL`
- `Gatekeeper structure check = FULL_PASS`

## 本次更新内容

### 1. Data Card Registry｜数据卡注册层

新增 Data Card schema 和 registry MVP，用于确保进入结论层的关键字段具备明确的数据来源、期间、单位、币种、来源等级、时间戳和结论使用权限。

核心原则：

> 没有数据卡，不得进入结论。

### 2. Earnings Basis Hardlock｜盈利口径硬锁

新增盈利口径检查，防止 GAAP actual、adjusted figures、management guidance、aggregator fields 等不同性质的数据被混用。

这可以降低 adjusted EPS、guidance EPS 或未回源聚合字段被误当成核心估值分母的风险。

### 3. Capital Structure Bridge｜股本结构桥接

新增股本结构桥接检查，用于校验 current shares、weighted diluted shares、market cap、EPS、PE 等字段之间是否自洽。

如果股本口径、EPS 分母或 PE 计算逻辑没有解释清楚，报告不能正式准出。

### 4. Data Integrity Hardlock Aggregator｜数据完整性聚合层

新增 hardlock 聚合层，将多个模块的检查结果汇总成统一的数据完整性准出判断。

只要关键 hardlock 模块失败，报告就不能正式发布。

### 5. Gatekeeper Integration｜Gatekeeper 接入硬锁结果

Gatekeeper 已接入 Data Integrity Hardlock verdict。

过去结构完整可能得到 `FULL_PASS`，但从本版本开始，结构完整不再等于正式可准出。如果 hardlock 缺失或失败，报告会被降级或阻断。

### 6. Valuation Metadata Guard｜估值工具元数据保护

估值工具仍然允许使用裸数字进行计算，但这类输出会被明确降级为：

```text
CALC_ONLY_NOT_FOR_CONCLUSION
```

只有具备完整 metadata 且被允许进入结论链的数据，才能支撑正式估值结论。

### 7. Industry Hard Fields MVP｜行业硬字段 MVP

新增 Industry Hard Fields MVP，首批支持两个行业适配器：

- `CN_SEMICONDUCTOR_GROWTH`
- `US_MANAGED_CARE`

行业关键字段必须存在，并且具备进入正式结论的权限。字段缺失、被阻断或仅可作为参考时，报告不能正式准出。

发布前，managed-care 适配器已从公司特定字段名泛化为行业通用分部字段，以降低行业泛化时的误阻断风险。
