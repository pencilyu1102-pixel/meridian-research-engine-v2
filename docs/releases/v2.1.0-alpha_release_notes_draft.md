# v2.1.0-alpha — Data Integrity Hardlock

## Summary

`v2.1.0-alpha` upgrades Meridian Research Engine from structure-level report gating to data-integrity-based release gating.

This release is not mainly about adding more report templates. Its core change is that whether a field is qualified to enter the conclusion layer is now enforced as a hard system gate.

## Why this release matters

Before `v2.1.0-alpha`, a report could look structurally complete while still carrying weak, mixed, or non-formal data inputs into the conclusion layer.

`v2.1.0-alpha` raises the standard. A conclusion is no longer accepted merely because the report is well-formed. It must also pass Data Integrity Hardlock.

## What changed

- Added Data Card schema and registry MVP.
- Added Earnings Basis Hardlock.
- Added Capital Structure Bridge MVP.
- Added Data Integrity Hardlock aggregator.
- Integrated Gatekeeper with Hardlock verdicts.
- Added valuation tool metadata guard.
- Added Industry Hard Fields MVP with `CN_SEMICONDUCTOR_GROWTH` and `US_MANAGED_CARE`.

## Hardlock layers

The release introduces four hardlock layers:

1. Data Card
2. Earnings Basis
3. Capital Structure Bridge
4. Industry Hard Fields

The guiding rules are:

```text
No card, no conclusion.
No basis, no formal valuation.
No bridge, no PE.
No industry hard fields, no FULL_PASS.
```

## Gatekeeper behavior change

Gatekeeper `FULL_PASS` is no longer treated as formally releasable by structure alone.

A report is only formally releasable when both conditions are satisfied:

- Data Integrity Hardlock = `PASS_FORMAL`
- Gatekeeper structure check = `FULL_PASS`

This separates structural completeness from formal release eligibility.

## Valuation tool metadata guard

Valuation tools can still calculate with bare numeric inputs, but those outputs are downgraded to `CALC_ONLY_NOT_FOR_CONCLUSION`.

This preserves calculation utility while preventing unsupported values from being promoted into formal conclusions.

## Industry Hard Fields

`v2.1.0-alpha` adds an MVP layer for industry-specific mandatory fields.

Supported adapters in this release:

- `CN_SEMICONDUCTOR_GROWTH`
- `US_MANAGED_CARE`

If required industry hard fields are missing, blocked, or only reference-level, the report cannot pass formal release gating.

Before alpha release, the `US_MANAGED_CARE` adapter was generalized from company-specific field names to generic managed care segment fields. This reduces false failures on peers such as non-reference managed care companies while preserving the formal hardlock requirement.

## Validation

Validation results at release-prep time:

- Full test suite: 59 passed.
- Key hardlock regression suite: 42 passed.

These checks cover Data Card, Earnings Basis, Capital Structure Bridge, Industry Hard Fields, Gatekeeper integration, and valuation metadata guard behavior.

## Public boundary

This release preparation preserves the public repository boundary:

- No real-company public sample reports.
- No real price ranges.
- No real valuation conclusions.
- No real trading suggestions.
- Synthetic identifiers only where examples are needed.

## Known limitations

- MVP uses sidecar JSON.
- Markdown body parsing is not included yet.
- No automatic market or financial data fetching.
- No full industry database.
- No real-company public sample reports.
